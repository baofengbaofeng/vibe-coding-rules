## 用户输入

```
$ARGUMENTS
```

## 目标

执行项目单元测试并生成覆盖率报告，包括：
1. 执行Maven单元测试
2. 生成JaCoCo覆盖率报告
3. 解析并输出各模块覆盖率统计

## 执行步骤

### 1. 确认项目环境

首先确认当前目录是Maven项目根目录，检查以下文件是否存在：
- `pom.xml`

如果不存在，输出错误信息并终止。

### 2. 执行单元测试

运行以下命令执行单元测试并生成覆盖率报告：

```
mvn clean test jacoco:report
```

如果用户指定了测试范围：
- 指定测试类：`mvn clean test -Dtest=指定的测试类 jacoco:report`
- 指定模块：`mvn clean test -Dtest=模块路径*Test jacoco:report`

捕获输出结果，分析测试通过/失败情况。

### 3. 解析覆盖率报告

读取JaCoCo生成的覆盖率报告文件：
- `target/site/jacoco/index.html`
- `target/site/jacoco/*.html`

解析各模块/包的覆盖率数据：
- 行覆盖率（Line Coverage）
- 分支覆盖率（Branch Coverage）
- 方法覆盖率（Method Coverage）
- 类覆盖率（Class Coverage）

### 5. 提供测试建议

根据覆盖率分析结果，提供以下建议：
- 未达标模块的测试补充建议
- 低覆盖率文件的具体测试场景建议
- 失败测试用例的修复建议

## 覆盖率要求参考

| 代码类型 | 行覆盖率要求 | 分支覆盖率要求 |
|---------|-------------|---------------|
| 业务核心代码（domain层） | >= 80% | >= 70% |
| Web层代码 | >= 60% | >= 50% |
| 工具类代码 | >= 70% | >= 60% |
| Entity/Model | >= 50% | - |

## 测试用例检查规则

根据 `rules/单元测试规范.md`，检查测试代码是否符合规范：

### 必须检查项
- 测试是否隔离外部依赖（无@SpringBootTest）
- 测试是否包含明确断言
- 测试方法命名是否规范

### 禁止事项检查
- 是否使用了@SpringBootTest
- 是否依赖外部系统（数据库、Redis等）
- 是否存在无断言的测试方法

## 使用示例

```
# 执行全部单元测试
/code-tests-check

# 执行指定测试类
/code-tests-check UserModuleTest

# 执行指定模块的测试
/code-tests-check domain
```

## 注意事项

1. 测试执行前会清理target目录
2. 覆盖率报告生成在 `target/site/jacoco/` 目录
3. 测试失败时会输出详细的失败信息
4. 建议在代码提交前执行此检查确保测试通过
5. 覆盖率数据仅供参考，测试质量更重要
