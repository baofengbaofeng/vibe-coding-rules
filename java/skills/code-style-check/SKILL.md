## 用户输入

```
$ARGUMENTS
```

## 目标

执行项目代码规范检查，包括：
1. Maven Checkstyle静态检查
2. 项目编码规范扫描（命名、注释、导入、异常处理等）
3. 输出详细的检查报告

## 执行步骤

### 1. 确认项目环境

首先确认当前目录是Maven项目根目录，检查以下文件是否存在：
- `pom.xml`
- `checkstyle.xml`

如果不存在，输出错误信息并终止。

### 2. 执行Maven Checkstyle检查

运行以下命令执行Checkstyle检查：

```
mvn validate
```

捕获输出结果，分析违规项。

### 3. 代码规范扫描

根据 `rules/项目编码规范.md` 定义的规范，扫描代码检查以下内容：

#### 禁止事项检查
- Tab缩进检查
- 行尾空格检查
- `System.out` / `System.err` 使用检查
- 裸 `Exception` / `Throwable` 捕获检查
- 星号导入检查
- 魔法数字检查

#### 命名规范检查
- 类命名后缀检查（Controller/Module/Entity/Mapper/Util等）
- 方法命名前缀检查（query/create/update/remove等）
- 常量命名检查（全大写下划线分隔）
- 变量命名检查（小驼峰）

#### 注释规范检查
- 类Javadoc检查
- 公共方法Javadoc检查
- 属性字段注释检查

#### 导入规范检查
- 导入顺序检查（java.* -> javax.* -> 第三方 -> 项目内部）
- 组间空行检查

## 检查规则参考

### 禁止事项

| 规则 | 说明 | 严重级别 |
|------|------|----------|
| 禁止Tab缩进 | 统一使用空格 | ERROR |
| 禁止行尾空格 | 行尾不能有空格 | ERROR |
| 禁止System.out | 使用SLF4J日志框架 | ERROR |
| 禁止裸Exception | 捕获具体异常类型 | ERROR |
| 禁止星号导入 | 每行导入一个类 | ERROR |
| 禁止魔法数字 | 定义为常量 | WARN |

### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 控制器 | XxxController | UserController |
| 领域模块 | XxxModule | UserModule |
| 数据实体 | XxxEntity | UserEntity |
| Mapper | XxxMapper | UserMapper |
| 工具类 | XxxUtil | StringUtil |
| 常量 | SCREAMING_SNAKE_CASE | MAX_RETRY_COUNT |

### 方法命名前缀

| 层级 | 操作 | 前缀 |
|------|------|------|
| Module | 查询 | query |
| Module | 创建 | create |
| Module | 更新 | update |
| Module | 删除 | remove |
| Mapper | 查询 | select |
| Mapper | 创建 | insert |
| Mapper | 更新 | update |
| Mapper | 删除 | delete |

## 使用示例

```
# 检查全项目
/code-style-check

# 检查指定文件
/code-style-check src/main/java/com/xxx/UserModule.java

# 检查指定模块
/code-style-check domain
```

## 注意事项

1. 检查前确保代码已编译通过
2. Checkstyle配置文件为 `checkstyle.xml`
3. 检查结果仅输出到命令行，不生成文件
4. 建议在代码提交前执行此检查
