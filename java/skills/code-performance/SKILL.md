## 用户输入

```
$ARGUMENTS
```

## 目标

执行项目代码性能风险检测，包括：
1. 数据库性能风险扫描
2. 缓存性能风险扫描
3. 并发控制风险扫描
4. 循环优化风险扫描
5. IO性能风险扫描
6. 内存管理风险扫描

## 执行步骤

### 1. 确认项目环境

首先确认当前目录是Maven项目根目录，检查以下文件是否存在：
- `pom.xml`
- `src/main/java` 目录

如果不存在，输出错误信息并终止。

### 2. 扫描代码文件

根据用户输入确定扫描范围：
- 无参数：扫描 `src/main/java` 下所有Java文件
- 指定文件：扫描指定文件
- 指定模块：扫描 `src/main/java/com/xxx/指定模块` 目录

### 3. 性能风险检测

根据 `rules/代码性能规范.md` 定义的规范，扫描以下性能风险：

#### 3.1 数据库性能风险

**高风险项**
- `SELECT *` 使用检查
- 循环内执行SQL操作
- WHERE条件缺失索引提示
- N+1查询模式

**检测模式**
```
// 检测 SELECT *
SELECT \*

// 检测循环内SQL
for\s*\([^)]*\)\s*\{[^}]*mapper\.(select|insert|update|delete)

// 检测N+1查询
for\s*\([^)]*\)\s*\{[^}]*query[^}]*\}
```

#### 3.2 缓存性能风险

**高风险项**
- Redis Key未设置过期时间
- KEYS命令使用（生产环境应使用SCAN）
- 大Key风险（Value过大）

**检测模式**
```
// 检测无过期时间的Redis操作
\.set\([^,]+,\s*[^)]+\)(?!.*expire)

// 检测KEYS命令
\.keys\(

// 检测大Key写入
\.set\([^,]+,\s*[^)]{10000,}\)
```

#### 3.3 并发控制风险

**高风险项**
- 共享变量无锁访问
- 直接创建线程（未使用线程池）
- ThreadLocal未清理
- 分布式锁超时设置缺失

**检测模式**
```
// 检测直接创建线程
new Thread\(

// 检测ThreadLocal未清理
ThreadLocal[^;]+;[^}]+(?!remove\(\))

// 检测锁无超时
\.lock\(\)(?!.*timeout)
```

#### 3.4 循环优化风险

**高风险项**
- 循环内数据库操作
- 循环内远程调用
- 循环内重复计算
- 循环内字符串拼接

**检测模式**
```
// 检测循环内字符串拼接
for\s*\([^)]*\)\s*\{[^}]*\+=

// 检测循环内远程调用
for\s*\([^)]*\)\s*\{[^}]*(remote|http|client)\.

// 检测循环内重复计算
for\s*\([^)]*\)\s*\{[^}]*calculate[^}]*\}
```

#### 3.5 IO性能风险

**高风险项**
- 资源未关闭（未使用try-with-resources）
- 未使用缓冲流
- 大文件一次性读取

**检测模式**
```
// 检测未关闭的资源
new (FileReader|FileWriter|FileInputStream|FileOutputStream)\([^)]+\)[^;]*;(?!\s*try)

// 检测非缓冲流
new (FileReader|FileWriter)\(

// 检测大文件读取
Files\.readAllBytes
```

#### 3.6 内存管理风险

**高风险项**
- 静态集合无限增长
- 监听器未注销
- 大对象未置空
- 集合未指定初始容量

**检测模式**
```
// 检测静态集合
private static final (List|Map|Set|Collection)

// 检测未指定容量的集合
new (ArrayList|HashMap|HashSet)\(\)
```

## 风险等级定义

| 等级 | 说明 | 影响 |
|------|------|------|
| 高 | 可能导致严重性能问题或系统故障 | 必须修复 |
| 中 | 可能导致性能下降或资源浪费 | 建议修复 |
| 低 | 优化建议，影响较小 | 可选修复 |

## 检测规则参考

### 数据库性能

| 风险项 | 检测模式 | 严重级别 |
|--------|----------|----------|
| 循环内SQL | for循环内调用mapper方法 | 高 |
| SELECT * | SQL语句使用SELECT * | 中 |
| N+1查询 | 循环内单条查询关联数据 | 高 |

### 缓存性能

| 风险项 | 检测模式 | 严重级别 |
|--------|----------|----------|
| Key无过期 | .set()后无expire | 高 |
| KEYS命令 | 调用.keys()方法 | 高 |
| 大Key | Value超过10KB | 中 |

### 并发控制

| 风险项 | 检测模式 | 严重级别 |
|--------|----------|----------|
| 直接创建线程 | new Thread() | 高 |
| ThreadLocal未清理 | ThreadLocal无remove调用 | 高 |
| 锁无超时 | lock()无timeout参数 | 中 |

### 循环优化

| 风险项 | 检测模式 | 严重级别 |
|--------|----------|----------|
| 循环内字符串拼接 | 循环内使用 += | 中 |
| 循环内远程调用 | 循环内调用remote/http | 高 |
| 循环内重复计算 | 循环内调用方法获取不变值 | 低 |

### IO性能

| 风险项 | 检测模式 | 严重级别 |
|--------|----------|----------|
| 资源未关闭 | IO流未在try-with-resources | 高 |
| 未用缓冲流 | 直接使用FileReader/FileWriter | 中 |

### 内存管理

| 风险项 | 检测模式 | 严重级别 |
|--------|----------|----------|
| 静态集合 | static final Collection | 高 |
| 未指定容量 | new ArrayList()无参数 | 低 |

## 使用示例

```
# 扫描全项目
/code-performance

# 扫描指定文件
/code-performance src/main/java/com/xxx/UserModule.java

# 扫描指定模块
/code-performance domain
```

## 注意事项

1. 本检测为静态代码扫描，无法检测运行时性能问题
2. 检测结果仅供参考，需结合实际业务场景判断
3. 建议在代码提交前执行此检测
4. 高风险项建议优先修复
