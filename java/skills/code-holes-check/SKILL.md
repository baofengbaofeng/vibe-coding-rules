## 用户输入

```
$ARGUMENTS
```

## 目标

执行项目代码安全风险检测，基于OWASP Top 10和项目安全规范，包括：
1. 注入防护检查（SQL/命令/XPath/JSONB注入）
2. 敏感信息泄露检查
3. 访问控制检查
4. XSS防护检查
5. 文件上传安全检查
6. 安全配置检查

## 执行步骤

### 1. 确认项目环境

首先确认当前目录是Maven项目根目录，检查以下文件是否存在：
- `pom.xml`
- `src/main/java` 目录

如果不存在，输出错误信息并终止。

### 2. 扫描代码文件

根据用户输入确定扫描范围：
- 无参数：扫描 `src/main/java` 下所有Java文件
- 指定文件：扫描指定文件
- 指定模块：扫描 `src/main/java/com/xxx/指定模块` 目录

同时扫描配置文件：
- `src/main/resources/application*.properties`
- `src/main/resources/application*.yml`
- MyBatis XML文件

### 3. 安全风险检测

根据 `rules/代码安全规范.md` 定义的规范，扫描以下安全风险：

#### 3.1 注入防护检查

**SQL注入风险**
```
// 检测 ${} 拼接（MyBatis）
\$\{[^}]+\}

// 检测字符串拼接SQL
"SELECT.*\+.*" 或 "INSERT.*\+.*" 或 "UPDATE.*\+.*" 或 "DELETE.*\+.*"

// 检测JdbcTemplate字符串拼接
jdbcTemplate\.(query|update)\([^)]*\+
```

**命令注入风险**
```
// 检测Runtime.exec
Runtime\.getRuntime\(\)\.exec\(

// 检测ProcessBuilder字符串拼接
new ProcessBuilder\(.*\+
```

**XPath注入风险**
```
// 检测XPath拼接
XPath.*\+.* 或 compile\(.*\+
```

#### 3.2 敏感信息泄露检查

**硬编码敏感信息**
```
// 检测硬编码密码
password\s*=\s*"[^"$]+"

// 检测硬编码密钥
(apiKey|api_key|secret|token)\s*=\s*"[^"$]+"

// 检测硬编码私钥
private.*key\s*=\s*"[^"$]+"
```

**日志敏感信息**
```
// 检测日志输出敏感字段
LOGGER\.(info|debug|warn|error).*\(.*password
LOGGER\.(info|debug|warn|error).*\(.*idCard
LOGGER\.(info|debug|warn|error).*\(.*phone
LOGGER\.(info|debug|warn|error).*\(.*bankCard
```

**配置文件敏感信息**
```
// 检测未加密的敏感配置
password\s*=\s*(?!ENCRYPT@)
secret\s*=\s*(?!ENCRYPT@)
token\s*=\s*(?!ENCRYPT@)
```

#### 3.3 访问控制检查

**无鉴权接口**
```
// 检测Controller方法是否需要鉴权
@GetMapping|@PostMapping|@PutMapping|@DeleteMapping
// 检查是否在白名单路径或是否有鉴权注解
```

**越权访问风险**
```
// 检测直接使用用户输入ID查询
selectById\(.*request\.
selectById\(.*parameter\.
```

#### 3.4 XSS防护检查

**输入校验缺失**
```
// 检测未校验的用户输入直接使用
@RequestParam.*String.*\)
@PathVariable.*String.*\)
// 检查是否有@Valid或手动校验
```

#### 3.5 文件上传安全检查

**文件类型校验缺失**
```
// 检测MultipartFile使用
MultipartFile\s+\w+
// 检查是否有类型校验
```

**文件名处理风险**
```
// 检测使用原始文件名
getOriginalFilename\(\)
// 检查是否有重命名处理
```

**路径遍历风险**
```
// 检测用户输入拼接路径
new File\(.*\+.*\)
Paths\.get\(.*\+.*\)
```

#### 3.6 安全配置检查

**CORS配置风险**
```
// 检测允许所有来源
addAllowedOrigin\("\*"\)
allowedOrigins.*\*
```

**监控端点暴露**
```
// 检测暴露所有端点
include:\s*"\*"
exposure.*\*
```

**危险功能启用**
```
// 检测Jackson多态反序列化
enableDefaultTyping
```

## 风险等级定义

| 等级 | 说明 | 影响 |
|------|------|------|
| 严重 | 可直接导致系统被攻击或数据泄露 | 必须立即修复 |
| 高危 | 存在较高安全风险，可能被利用 | 必须修复 |
| 中危 | 存在一定安全风险 | 建议修复 |
| 低危 | 安全改进建议 | 可选修复 |

## 检测规则参考

### 注入防护

| 风险项 | 检测模式 | 严重级别 |
|--------|----------|----------|
| SQL注入 | ${} 拼接 | 严重 |
| SQL注入 | 字符串拼接SQL | 严重 |
| 命令注入 | Runtime.exec拼接 | 严重 |
| XPath注入 | XPath字符串拼接 | 高危 |

### 敏感信息

| 风险项 | 检测模式 | 严重级别 |
|--------|----------|----------|
| 硬编码 | password/secret/token硬编码 | 高危 |
| 配置未加密 | 敏感配置未使用ENCRYPT@ | 高危 |
| 日志泄露 | 日志输出敏感字段 | 中危 |

### 访问控制

| 风险项 | 检测模式 | 严重级别 |
|--------|----------|----------|
| 鉴权缺失 | 接口未配置鉴权 | 高危 |
| 越权访问 | 直接使用用户ID查询 | 中危 |

### 文件上传

| 风险项 | 检测模式 | 严重级别 |
|--------|----------|----------|
| 类型校验缺失 | MultipartFile无类型检查 | 高危 |
| 文件名风险 | 使用原始文件名 | 中危 |
| 路径遍历 | 用户输入拼接路径 | 高危 |

### 安全配置

| 风险项 | 检测模式 | 严重级别 |
|--------|----------|----------|
| CORS | 允许所有来源 | 中危 |
| 端点暴露 | 暴露所有端点 | 中危 |
| 危险功能 | enableDefaultTyping | 高危 |

## 使用示例

```
# 扫描全项目
/code-holes-check

# 扫描指定文件
/code-holes-check src/main/java/com/xxx/UserService.java

# 扫描指定模块
/code-holes-check web
```

## 注意事项

1. 本检测为静态代码扫描，无法检测运行时安全问题
2. 检测结果仅供参考，需结合实际业务场景判断
3. 建议在代码提交前执行此检测
4. 严重和高危问题必须修复后方可合入
5. 定期执行检测，确保代码安全
