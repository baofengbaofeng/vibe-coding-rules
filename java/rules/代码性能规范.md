# 代码性能规范.md

本规范定义代码编写过程中的性能优化标准，涵盖数据库操作、缓存使用、并发控制、循环处理、IO操作等方面，确保系统高效稳定运行。

---

## 数据库性能

### 查询优化

- **禁止全表扫描**：WHERE条件必须命中索引
- **禁止`SELECT *`**：必须指定需要的字段列表
- **复杂查询需EXPLAIN分析**：执行计划中type避免出现ALL
- **分页查询必须带排序**：避免分页结果不稳定

```
// 禁止
SELECT * FROM user WHERE name = 'test'

// 推荐
SELECT id, name, status FROM user WHERE name = 'test' LIMIT 100
```

### 索引规范

- 避免在索引列上使用函数或计算
- 单表索引数量不超过5个
- 联合索引遵循最左前缀原则

```
// 禁止：索引列使用函数
WHERE DATE(create_time) = '2026-04-17'

// 推荐：范围查询
WHERE create_time >= '2026-04-17 00:00:00' AND create_time < '2026-04-18 00:00:00'
```

### 批量操作

- **禁止循环执行SQL**：必须使用批量操作
- 批量插入单次不超过1000条
- 批量更新使用CASE WHEN或ON DUPLICATE KEY

```
// 禁止：循环插入
for (UserEntity user : userList) {
    userMapper.insert(user);
}

// 推荐：批量插入
userMapper.batchInsert(userList);
```

## 缓存性能

### Redis使用规范

- **大Key拆分**：单个Key的Value不超过10KB
- **Key必须设置过期时间**：避免内存泄漏
- **Key必须设置业务前缀**：如 `project:dev:user:1001`
- **禁止使用KEYS命令**：生产环境使用SCAN

```
// 推荐：带前缀和过期时间
String key = "project:dev:user:" + userId;
redisTemplate.opsForValue().set(key, value, 30, TimeUnit.MINUTES);
```

### 缓存策略

- **缓存穿透**：查询空结果也缓存，设置较短过期时间
- **缓存击穿**：热点数据使用分布式锁或永不过期
- **缓存雪崩**：过期时间加随机值，避免同时失效

```
// 缓存穿透防护
public UserEntity getUserById(Long id) {
    String key = "project:user:" + id;
    UserEntity user = redisTemplate.opsForValue().get(key);
    if (user != null) {
        return user;
    }

    user = userMapper.selectById(id);
    if (user == null) {
        // 空结果也缓存，设置较短过期时间
        redisTemplate.opsForValue().set(key, new UserEntity(), 5, TimeUnit.MINUTES);
        return null;
    }

    redisTemplate.opsForValue().set(key, user, 30, TimeUnit.MINUTES);
    return user;
}
```

## 并发控制

### 线程安全

- **原子操作使用Atomic类**：如AtomicInteger、AtomicReference
- **共享变量必须加锁**：使用synchronized或ReentrantLock
- **优先使用线程本地变量**：ThreadLocal避免竞争

```
// 推荐：原子操作
private final AtomicInteger counter = new AtomicInteger(0);
counter.incrementAndGet();

// 推荐：线程本地变量
private static final ThreadLocal<SimpleDateFormat> dateFormat =
    ThreadLocal.withInitial(() -> new SimpleDateFormat("yyyy-MM-dd"));
```

### 分布式锁

- **必须使用Redisson实现**：禁止自行实现分布式锁
- **必须设置超时时间**：避免死锁
- **锁粒度要小**：避免大范围锁定

```
// 推荐：Redisson分布式锁
RLock lock = redissonClient.getLock("project:lock:" + resourceId);
try {
    if (lock.tryLock(10, 30, TimeUnit.SECONDS)) {
        // 执行业务逻辑
    }
} finally {
    if (lock.isHeldByCurrentThread()) {
        lock.unlock();
    }
}
```

### 线程池

- **禁止直接创建线程**：必须使用线程池
- **合理配置线程池参数**：核心线程数、最大线程数、队列容量、拒绝策略
- **线程池必须有名称**：便于监控和排查问题
- **必须包含关闭钩子**：以便系统关闭或重启时对队列内的数据进行处理

```
// 推荐：配置线程池
@Bean("asyncExecutor")
public Executor asyncExecutor() {
    ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
    executor.setCorePoolSize(10);
    executor.setMaxPoolSize(20);
    executor.setQueueCapacity(100);
    executor.setThreadNamePrefix("async-task-");
    executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
    executor.initialize();
    return executor;
}
```

## 循环优化

### 循环内禁止操作

- **禁止循环内执行数据库操作**：提取到循环外批量处理
- **禁止循环内调用远程接口**：改为批量接口或并行调用
- **禁止循环内创建大量对象**：考虑对象复用

```
// 禁止：循环内查库
for (Long userId : userIds) {
    UserEntity user = userMapper.selectById(userId);
    // 处理逻辑
}

// 推荐：批量查询
List<UserEntity> users = userMapper.selectByIds(userIds);
Map<Long, UserEntity> userMap = users.stream()
    .collect(Collectors.toMap(UserEntity::getId, Function.identity()));
for (Long userId : userIds) {
    UserEntity user = userMap.get(userId);
    // 处理逻辑
}
```

### 循环优化技巧

- **减少循环次数**：提前过滤无效数据
- **避免循环内重复计算**：提取到循环外
- **合理使用Stream API**：注意并行流的线程安全

```
// 禁止：循环内重复计算
for (UserEntity user : users) {
    if (user.getCreateTime().after(calculateDeadline())) {
        // 处理逻辑
    }
}

// 推荐：提取计算
Date deadline = calculateDeadline();
for (UserEntity user : users) {
    if (user.getCreateTime().after(deadline)) {
        // 处理逻辑
    }
}
```

### 集合遍历

- **遍历时避免修改集合**：使用Iterator或创建副本
- **根据场景选择合适集合**：ArrayList遍历快，LinkedList插入快
- **大集合分批处理**：避免内存溢出

```
// 推荐：分批处理大集合
List<List<UserEntity>> batches = Lists.partition(users, 1000);
for (List<UserEntity> batch : batches) {
    processBatch(batch);
}
```

## IO性能

### 文件操作

- **使用缓冲流**：BufferedReader/BufferedWriter
- **必须关闭资源**：使用try-with-resources
- **避免频繁打开关闭**：批量操作时复用连接

```
// 推荐：缓冲流 + try-with-resources
try (BufferedReader reader = new BufferedReader(new FileReader(file))) {
    String line;
    while ((line = reader.readLine()) != null) {
        // 处理逻辑
    }
}
```

### 网络请求

- **设置合理超时**：连接超时、读取超时
- **使用连接池**：HTTP连接池复用
- **批量操作并行处理**：CompletableFuture并行调用

```
// 推荐：并行调用
List<CompletableFuture<Result>> futures = requests.stream()
    .map(req -> CompletableFuture.supplyAsync(() -> remoteService.call(req), executor))
    .collect(Collectors.toList());
List<Result> results = futures.stream()
    .map(CompletableFuture::join)
    .collect(Collectors.toList());
```

### 序列化

- **选择高效序列化方式**：JSON通用性好，Protobuf性能高
- **避免序列化大对象**：按需传输字段
- **禁用循环引用检测**：FastJSON设置DisableCircularReferenceDetect

## 内存管理

### 对象创建

- **避免频繁创建短生命周期对象**：考虑对象池
- **大对象及时置空**：帮助GC回收
- **字符串拼接使用StringBuilder**：循环场景避免String拼接

```
// 禁止：循环内字符串拼接
String result = "";
for (String item : items) {
    result += item;
}

// 推荐：StringBuilder
StringBuilder sb = new StringBuilder();
for (String item : items) {
    sb.append(item);
}
String result = sb.toString();
```

### 集合初始化

- **指定初始容量**：避免频繁扩容
- **及时清理无用集合**：调用clear()或置null

```
// 推荐：指定初始容量
List<UserEntity> users = new ArrayList<>(expectedSize);
Map<Long, UserEntity> userMap = new HashMap<>(expectedSize);
```

### 避免内存泄漏

- **ThreadLocal必须清理**：finally中调用remove()
- **监听器必须注销**：避免持有对象引用
- **静态集合谨慎使用**：长期持有导致无法GC

```
// 推荐：ThreadLocal清理
try {
    threadLocal.set(value);
    // 业务逻辑
} finally {
    threadLocal.remove();
}
```

## 日志性能

### 日志级别

- **生产环境使用INFO级别**：避免DEBUG日志影响性能
- **日志级别判断**：避免不必要的字符串拼接

```
// 禁止：无条件拼接
LOGGER.debug("User info: " + user.toString());

// 推荐：级别判断
if (LOGGER.isDebugEnabled()) {
    LOGGER.debug("User info: {}", user.toString());
}
```

### 日志内容

- **禁止打印大对象**：避免序列化开销
- **禁止循环内打印日志**：改为批量或汇总日志
- **异常日志必须打印堆栈**：便于问题定位

### 慢查询监控

- 数据库慢查询阈值：1秒
- 接口响应阈值：500毫秒
- 定期分析慢查询日志

### 性能测试

- 核心接口必须进行压测
- 压测通过标准：QPS、RT、错误率
- 上线前进行全链路压测

### 检测工具

- **虚拟机监控**：JConsole、VisualVM、Arthas
- **数据库监控**：Slow Query Log、Explain
