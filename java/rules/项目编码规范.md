# 项目编码规范.md

这份编码规范基于项目的实际代码分析总结，涵盖了命名、注释、导入、注解、异常处理、日志、测试等多个方面，可作为该项目的编码标准。

## 禁止事项
1.  **禁止使用 Tab 缩进**：统一使用空格
2.  **禁止行尾留有空格**
3.  **禁止使用 `System.out` 或 `System.err` 调试**：必须使用日志框架（SLF4J）
4.  **禁止捕获裸 `Exception` 或 `Throwable`**：除非是在最顶层的异常处理器中
5.  **禁止循环依赖**：包括类之间和 Maven 模块之间的循环引用
6.  **禁止使用星号导入**：禁止 `import java.util.*;`
7.  **禁止魔法数字**：必须定义为 `public static final` 或 `private static final` 常量
8.  **禁止使用反射/元编程**：除非是框架级开发且经过评审
9.  **禁止交叉调用**：禁止跨Module调用其他领域的Mapper代码

## 导入规范

禁止通配符导入，必须按以下顺序分组，每组内部按字母序排列，组间空一行：
1. `java.*` (标准库)
2. `javax.*` (标准库扩展)
3. 第三方库 (org.*, com.google.* 等)
4. 项目内部包 (`com.agent.*`)

**导入注释**：每行只能导入一个类。建议在复杂的导入后添加用途简述，确保整行长度在 60-120 字符之间

## 命名规范

- **原则**: 严禁二义性缩写。例如使用 `userIdentifier` 而非 `uid`
- **泛型变量**: 单个大写字母 (T, E, K, V)

### Class命名
- **大驼峰命名法**（PascalCase），类名后缀表示类型：
  - 控制器：`Controller`，如`UserController`
  - 领域接口：`Service`，如`UserService`
  - 领域模块：`Module`，如`UserModule`
  - 数据实体：`Entity`，如`UserEntity`
  - 中间对象：`Model`，如`UserModel`
  - Mapper：`Mapper`，如`UserMapper`
  - 工具类：`Util`，如`StringCryptoUtil`
  - 异常类：`Exception`，如`PermitValidateException`
  - 配置类：`Config`，如`UserPermitConfig`
  - 切面类：`Aspect`，如`UserPermitAspect`
  - 拦截器：`Interceptor`，如`UserPermitInterceptor`
  - 枚举类：`Enum` 或 `Type`，如`UserStatusEnum`
  - 请求对象：`Request`，如`UserCreateRequest`
  - 响应对象：`Response`，如`UserCreateResponse`

### 方法命名
- **小驼峰命名法**（camelCase），动词前缀表示操作类型：
  - `build`：构建，如 `buildFileDownloadUrl`
  - `convert`：转换，如 `convertUserProfileEntityToCustomerEntity`
  - 业务领域层操作：
    - `query`：查询单行数据和多行数据，如 `queryUserProfileById` 或 `queryUserProfileList`
    - `create`：创建操作，如 `createUserProfile`
    - `update`：更新操作，如 `updateUserProfile`
    - `remove`：移除操作，如 `removeUserProfileById`
  - 数据访问层操作：
    - `select`：查询单行数据和多行数据，如 `selectUserProfileById` 或 `selectUserProfileList`
    - `insert`：创建操作，如 `insertUserProfile`
    - `update`：更新操作，如 `updateUserProfile`
    - `delete`：进行删除，如 `deleteUserProfileById`
  - 控制器层操作：
    - `createXXXActionHandle`：创建操作，如 `createUserProfileActionHandle`
    - `updateXXXActionHandle`：编辑操作，如 `updateUserProfileActionHandle`
    - `removeXXXActionHandle`：删除操作，如 `removeUserProfileActionHandle`
    - `queryXXXXActionHandle`：查询单行数据和多行数据，如 `queryUserProfileByIdActionHandle` 或 `queryUserProfileListActionHandle`

### 变量命名
- **小驼峰命名法**（camelCase），使用有意义的名称，避免单字母变量（循环变量除外）

### 常量命名
- **全大写，下划线分隔**（SCREAMING_SNAKE_CASE）

### 枚举命名
- 枚举类以 `Enum` 或 `Type` 结尾，枚举值全大写

### 类型与语法
- **Java 8 特性**:
  - 优先使用 `Optional<T>` 处理可能为空的返回值，禁止返回 `null`
  - 集合操作优先使用 `Stream API`
  - 涉及时间处理必须使用 `java.time` 包 (LocalDateTime)，禁止使用 `java.util.Date`
- **Spring Boot**:
  - 统一使用构造函数注入（Constructor Injection），禁止使用 `@Autowired` 在字段上直接注入
  - 异步操作必须在配置类中显式定义 `Executor` 并使用 `@Async("beanName")`

### 文档与注释
- **类与公共方法**: 必须使用标准 Javadoc (`/** ... */`)。
- **Javadoc**: 包含 `@param`, `@return`, `@throws`。每行描述文字长度必须不少于 60 个字符
- **行内注释**: 属性字段必须提供注释，注释整行长度（代码+注释）应保持在 60-120 字符

### 异常处理
- **具体化**: 必须捕获具体的异常类型（如 `IOException`），禁止 `catch (Exception e)` 或 `catch (Throwable e)`
- **异常链**: 抛出新异常时必须包含原始异常：`throw new CheckedException("...", e);`
- **资源释放**: 必须使用 `try-with-resources` 语句处理 IO、数据库连接等
- **全局处理**: 使用 `@RestControllerAdvice` 统一处理业务异常与系统异常

### 日志规范
- **定义**: `private static final Logger LOGGER = LoggerFactory.getLogger(xxx.class);`
- **级别**: 严格区分 DEBUG, INFO, WARN, ERROR。禁止在生产环境下打印堆栈轨迹

### Lombok
- 实体类标准注解
  ```
  @Getter
  @Setter
  @AllArgsConstructor
  @Builder // 或 @SuperBuilder（用于继承场景）
  @NoArgsConstructor
  @ToString
  public class PageResult<T> implements Serializable {
      // ...
  }
  ```
  
- 工具和常量类注解
  ```
  @NoArgsConstructor(access = lombok.AccessLevel.PRIVATE)
  public final class GeneralExceptionMessage {
      // ...
  }
  ```

## 数据库规范

### 基础约束
- **引擎**: 强制使用 `InnoDB`。
- **字符集**: `utf8mb4`。
- **规范**:
  - 每个表必须有自增主键、创建人、创建时间、更新人、更新时间、删除标记，其中删除标记字段保存时间戳，大于0则代表被删除。
    - `id bigint(20) unsigned NOT NULL DEFAULT '0' COMMENT '数据库主键ID'`
    - `creator varchar(45) NOT NULL DEFAULT '' COMMENT '创建人'`
    - `create_time timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'`
    - `updater varchar(45) NOT NULL DEFAULT '' COMMENT '创建人'`
    - `update_time timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '创建时间'`
    - `deleted bigint(20) unsigned NOT NULL DEFAULT '0' COMMENT '删除标记'`,
  - 禁止使用 `SELECT *`，必须指定字段列表。
  - 禁止在循环中执行 SQL 操作，必须使用 MyBatis 批量操作。
  - 禁止使用外键。

### 性能优化
- **索引**:
  - 索引数量单表不超过 5 个。
  - 严禁全表扫描，`WHERE` 条件必须命中索引。
  - 复杂查询必须通过 `EXPLAIN` 分析。
