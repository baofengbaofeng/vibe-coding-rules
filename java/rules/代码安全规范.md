# 代码安全规范.md

本规范基于 OWASP Top 10，聚焦当前项目（Java 8 / Spring Boot / MyBatis / MySQL）的实际威胁面，定义安全开发标准。

---

## 注入防护

### SQL注入防护

使用 MyBatis 进行数据库操作时必须使用参数绑定，禁止字符串拼接：

```
<!-- ❌ 错误：使用 ${} 拼接，存在注入风险 -->
SELECT * FROM archive WHERE employee_code = '${employeeCode}'

<!-- ✅ 正确：使用 #{} 参数绑定 -->
SELECT * FROM archive WHERE employee_code = #{employeeCode}
```

**动态表名/列名场景：** 必须在代码层白名单校验后再拼接。

```
// ❌ 错误：直接拼接表名
String sql = "SELECT * FROM " + tableName;

// ✅ 正确：白名单校验
private static final Set<String> ALLOWED_TABLES = Set.of("user", "order", "archive");
if (!ALLOWED_TABLES.contains(tableName)) {
    throw new IllegalArgumentException("Invalid table name");
}
```

使用 JdbcTemplate 进行数据库操作时，避免直接拼接 SQL 字符串：

```
// ❌ 错误：使用字符串拼接，存在注入风险
String sql = "SELECT * FROM archive WHERE employee_code = '" + employeeCode + "'";

// ✅ 正确：使用占位符参数绑定
String sql = "SELECT * FROM archive WHERE employee_code = ?";
jdbcTemplate.query(sql, new Object[]{employeeCode}, rowMapper);
```

### 命令注入防护

禁止使用 `Runtime.exec(String command)` 或 `ProcessBuilder.command(String... command)` 执行包含用户输入直接/间接拼装的系统命令字符串。

```
// ❌ 错误：直接拼接用户输入
Runtime.getRuntime().exec("ls " + userInput);

// ✅ 正确：使用参数数组，避免shell解析
ProcessBuilder pb = new ProcessBuilder("ls", validatedPath);
```

### JSONB 注入

```
// ❌ 错误：拼接 JSONB 查询条件
String sql = "SELECT * FROM config WHERE data @> '{\"type\": \"" + type + "\"}'::jsonb";

// ✅ 安全：参数化 JSONB 的查询
String sql = "SELECT * FROM config WHERE data @> #{jsonCondition}::jsonb";
```

### XPath注入防护

如果使用XPath查询XML，必须参数化处理：

```
// ❌ 错误：拼接XPath
String xpath = "//user[@name='" + username + "']";

// ✅ 正确：使用变量解析
XPathExpression expr = xpath.compile("//user[@name=$name]");
expr.setVariable(new QName("name"), username);
```

## 敏感信息

### 配置信息加密

所有的敏感配置项都必须使用 KMS 进行加密，避免泄露：

```
# ✅ 正确：使用加密配置
spring.datasource.password = ENCRYPT@加密密文
spring.redis.password = ENCRYPT@加密密文
```

**必须加密的配置项**：
- 数据库连接密码
- Redis连接密码
- KMS密钥
- 第三方API Token
- HTTP Basic认证密码
- 私钥文件路径和密码

### 敏感字段加密存储

身份证、银行卡、手机号码等敏感字段使用 `@PersistenceCryptoField` 加密存储：

```
@PersistenceCryptoClazz
public class EmployeeEntity {

    @PersistenceCryptoField
    private String idCardNumber;     // 身份证号加密存储

    @PersistenceCryptoField
    private String phoneNumber;      // 手机号加密存储

    @PersistenceCryptoField
    private String bankCardNumber;   // 银行卡号加密存储
}
```

### 禁止硬编码敏感信息

```
// ❌ 错误：直接在代码里硬编码敏感信息
private String apiKey = "sk-xxxxx";
private String password = "admin123";

// ✅ 正确：从配置文件读取并加密存储
@Value("${app.security.api-key}")
private String apiKey;
```

### 日志脱敏

日志中禁止输出敏感信息，必须进行脱敏处理：

```
// ❌ 错误：直接输出敏感信息
LOGGER.info("User phone: {}", user.getPhoneNumber());

// ✅ 正确：脱敏后输出
LOGGER.info("User phone: {}", maskPhoneNumber(user.getPhoneNumber()));

// 脱敏工具方法
public static String maskPhoneNumber(String phone) {
    if (StringUtils.isBlank(phone) || phone.length() < 7) {
        return "***";
    }
    return phone.substring(0, 3) + "****" + phone.substring(phone.length() - 4);
}
```

**必须脱敏的字段**：
- 身份证号（保留前3后4位）
- 手机号（保留前3后4位）
- 银行卡号（保留前4后4位）
- 密码（完全不输出）
- 密钥（完全不输出）

## 访问控制

### 请求验证

除系统监控相关的接口外，其他API都需要进行访问验证：

- **服务端调用API**：必须经过 `RequestAccessInterceptor` 拦截器验证（HTTP Basic认证）
- **前端调用API**：必须经过 `RequestPermitInterceptor` 拦截器验证（权限校验）
- **禁止新增无鉴权接口**

```
// 白名单接口配置示例
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(requestPermitInterceptor())
                .addPathPatterns("/**")
                .excludePathPatterns("/health", "/actuator/**", "/error");
    }
}
```

### 权限检查

业务层必须进行数据权限校验，避免越权访问：

```
public void updateArchive(Long archiveId, ArchiveUpdateRequest request) {
    // 获取当前用户的数据权限维度
    OperationPermitDims permitDims = operationPermitCalculate
            .getOperationPermitDims(roleId, funcId);

    // 查询档案时校验数据权限
    ArchiveEntity archive = archiveMapper.selectById(archiveId);
    if (!hasDataPermission(archive, permitDims)) {
        throw new BasaltCheckedException(GeneralExceptionMessage.NO_PERMISSION);
    }

    // 执行更新操作
    archiveMapper.updateById(archive);
}
```

### 操作审计

所有请求通过 `RequestAuditJournalPrint` 记录审计日志，字段包括：
- 用户身份（userName、userIp）
- 操作信息（url、method、params）
- 响应结果（status、response）

## 安全配置

### 监控端点

生产环境必须关闭敏感端点：

```
# ✅ 正确：仅暴露必要端点
management:
  endpoints:
    web:
      exposure:
        include: health,info
  endpoint:
    health:
      show-details: never
```

```
# ❌ 错误：暴露所有端点
management:
  endpoints:
    web:
      exposure:
        include: "*"
```

### 跨域配置

禁止允许所有来源，必须使用白名单：

```
// ❌ 错误：允许所有来源
config.addAllowedOrigin("*");
```

```
// ✅ 正确：白名单来源
@Configuration
public class CorsConfig {

    @Value("${app.cors.allowed-origins}")
    private List<String> allowedOrigins;

    @Bean
    public CorsFilter corsFilter() {
        CorsConfiguration config = new CorsConfiguration();
        allowedOrigins.forEach(config::addAllowedOrigin);
        config.setAllowCredentials(true);
        config.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE"));
        config.setAllowedHeaders(Collections.singletonList("*"));
        config.setMaxAge(3600L);

        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", config);
        return new CorsFilter(source);
    }
}
```

### 安全响应头

添加安全相关的HTTP响应头：

```
@Configuration
public class SecurityHeadersConfig {

    @Bean
    public FilterRegistrationBean<HeaderFilter> headerFilter() {
        FilterRegistrationBean<HeaderFilter> registration = new FilterRegistrationBean<>();
        registration.setFilter(new HeaderFilter());
        registration.addUrlPatterns("/*");
        return registration;
    }
}

public class HeaderFilter implements Filter {
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws IOException, ServletException {
        HttpServletResponse httpResponse = (HttpServletResponse) response;
        httpResponse.setHeader("X-Content-Type-Options", "nosniff");
        httpResponse.setHeader("X-Frame-Options", "DENY");
        httpResponse.setHeader("X-XSS-Protection", "1; mode=block");
        httpResponse.setHeader("Strict-Transport-Security", "max-age=31536000");
        chain.doFilter(request, response);
    }
}
```

### Cookie安全

Cookie必须设置安全属性：

```
// ✅ 正确：设置Cookie安全属性
Cookie cookie = new Cookie("sessionId", sessionId);
cookie.setHttpOnly(true);    // 防止JS读取
cookie.setSecure(true);      // HTTPS传输
cookie.setPath("/");
cookie.setMaxAge(3600);
response.addCookie(cookie);
```

## 依赖安全

### 依赖版本管理

- 定期检查依赖是否有已知漏洞（关注CVE公告）
- 使用 `mvn dependency:analyze` 检查未使用的依赖
- 禁止引入不维护的依赖库

```
<!-- 使用OWASP Dependency Check插件 -->
<plugin>
    <groupId>org.owasp</groupId>
    <artifactId>dependency-check-maven</artifactId>
    <version>8.2.1</version>
    <executions>
        <execution>
            <goals>
                <goal>check</goal>
            </goals>
        </execution>
    </executions>
</plugin>
```

### 禁用危险功能

```
// ❌ 禁止：启用Jackson多态反序列化
objectMapper.enableDefaultTyping();

// ✅ 正确：禁用危险功能
objectMapper.disable(SerializationFeature.FAIL_ON_EMPTY_BEANS);
objectMapper.disable(DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES);
```
