# 项目编码规范

## 概述
本文档定义Rust项目的编码风格、命名规范、注释要求等基础编码标准，适用于所有Rust源代码文件。

---

## 代码风格

### 1. 格式化
- 使用`cargo fmt`统一代码格式，配置使用默认Rust风格
- 每行代码不超过100个字符
- 使用4个空格缩进，禁止使用Tab

### 2. 导入顺序
```
// 标准库
use std::collections::HashMap;
use std::fmt;

// 外部依赖
use serde::{Deserialize, Serialize};
use tokio::sync::Mutex;

// 内部模块
use crate::domain::user::model::User;
use crate::infrastructure::database::connection::DatabasePool;
```

### 3. 模块组织
```
// 模块声明顺序
mod submodule;

// 导入声明
use ...

// 类型定义
pub struct ...

// 常量定义
const ...

// 静态变量
static ...

// 函数定义
impl ...
```

## 命名规范

### 1. 文件命名
- 使用小写字母和下划线：`user_service.rs`
- 测试文件：`user_service_test.rs` 或 `test_user_service.rs`
- 模块文件：`mod.rs`（旧风格）或与目录同名（新风格）

### 2. 模块命名
- 使用小写字母和下划线：`user_management`
- 避免缩写，使用完整单词

### 3. 类型命名
| 类型 | 规范 | 示例 |
|------|------|------|
| 结构体 | 大驼峰 | `UserProfile`, `DatabaseConfig` |
| 枚举 | 大驼峰 | `UserRole`, `ErrorCode` |
| trait | 大驼峰 | `Repository`, `Service` |
| 类型别名 | 大驼峰 | `UserId`, `Result<T>` |

### 4. 变量和函数命名
| 类型 | 规范 | 示例 |
|------|------|------|
| 变量 | 蛇形命名 | `user_name`, `max_retries` |
| 函数 | 蛇形命名 | `get_user_by_id`, `validate_input` |
| 常量 | 全大写+下划线 | `MAX_RETRY_COUNT`, `DEFAULT_TIMEOUT` |
| 静态变量 | 全大写+下划线 | `APP_NAME`, `VERSION` |

### 5. 生命周期参数
- 使用短小有意义的名称：`'a`, `'ctx`, `'db`
- 避免使用`'static`除非必要

### 6. 泛型参数
- 使用大写字母：`T`, `E`, `K`, `V`
- 有意义的名称：`Item`, `Error`, `Key`, `Value`

## 注释规范

### 1. 文档注释
```
/// 获取用户信息
///
/// # 参数
/// - `user_id`: 用户ID
///
/// # 返回值
/// - `Ok(User)`: 用户信息
/// - `Err(Error)`: 错误信息
///
/// # 示例
/// ```
/// let user = get_user(123).await?;
/// ```
pub async fn get_user(user_id: i64) -> Result<User, Error> {
    // ...
}
```

### 2. 模块注释
```
//! 用户管理模块
//!
//! 提供用户相关的业务逻辑，包括：
//! - 用户注册
//! - 用户登录
//! - 用户信息管理
```

### 3. 行内注释
```
// 检查用户权限
if !user.has_permission(Permission::Read) {
    return Err(Error::PermissionDenied);
}

// TODO: 优化查询性能
let users = repository.find_all().await?;
```

### 4. 注释标签
- `TODO`: 待完成的功能
- `FIXME`: 需要修复的问题
- `NOTE`: 重要说明
- `WARNING`: 警告信息
- `DEPRECATED`: 已弃用的代码

## 类型定义规范

### 1. 结构体定义
```
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct User {
    /// 用户ID
    pub id: i64,
    
    /// 用户名
    pub username: String,
    
    /// 邮箱地址
    pub email: String,
    
    /// 创建时间
    #[serde(with = "chrono::serde::ts_seconds")]
    pub created_at: DateTime<Utc>,
}
```

### 2. 枚举定义
```
#[derive(Debug, Clone, Copy, Serialize, Deserialize)]
pub enum UserRole {
    /// 普通用户
    User,
    
    /// 管理员
    Admin,
    
    /// 超级管理员
    SuperAdmin,
}

#[derive(Debug, thiserror::Error)]
pub enum AppError {
    #[error("数据库错误: {0}")]
    Database(#[from] sqlx::Error),
    
    #[error("验证错误: {0}")]
    Validation(String),
    
    #[error("未授权访问")]
    Unauthorized,
}
```

### 3. trait定义
```
/// 用户仓储接口
#[async_trait]
pub trait UserRepository: Send + Sync {
    /// 根据ID查找用户
    async fn find_by_id(&self, id: i64) -> Result<Option<User>, Error>;
    
    /// 保存用户
    async fn save(&self, user: &User) -> Result<(), Error>;
    
    /// 删除用户
    async fn delete(&self, id: i64) -> Result<(), Error>;
}
```

## 函数定义规范

### 1. 函数签名
```
pub async fn create_user(
    pool: &DatabasePool,
    username: &str,
    email: &str,
    password: &str,
) -> Result<User, AppError> {
    // 函数体
}
```

### 2. 参数顺序
1. 上下文参数（数据库连接、配置等）
2. 输入参数
3. 可选参数

### 3. 返回值
- 使用`Result<T, E>`处理可能失败的操作
- 错误类型应该明确具体
- 避免返回`Option<T>`除非确实可能为空

### 4. 异步函数
- 使用`async fn`定义异步函数
- 为trait中的异步方法使用`#[async_trait]`
- 合理使用`.await`，避免阻塞

## 错误处理规范

### 1. 错误类型定义
```
#[derive(Debug, thiserror::Error)]
pub enum ServiceError {
    #[error("用户不存在: {0}")]
    UserNotFound(i64),
    
    #[error("邮箱已存在: {0}")]
    EmailExists(String),
    
    #[error("密码错误")]
    InvalidPassword,
    
    #[error("内部错误: {0}")]
    Internal(#[from] anyhow::Error),
}
```

### 2. 错误处理
```
// 使用?操作符传播错误
let user = repository.find_by_id(user_id).await?;

// 匹配特定错误
match result {
    Ok(user) => Ok(user),
    Err(ServiceError::UserNotFound(_)) => {
        // 处理用户不存在的情况
        Err(AppError::NotFound)
    }
    Err(e) => Err(AppError::from(e)),
}

// 转换错误类型
repository.save(&user)
    .await
    .map_err(|e| AppError::Database(e.into()))?;
```

### 3. 禁止行为
- 禁止在生产代码中使用`unwrap()`或`expect()`
- 禁止忽略错误（使用`let _ = ...`）
- 禁止使用`panic!()`除非是程序无法恢复的错误

## 性能优化规范

### 1. 避免不必要的克隆
```
// 不好：不必要的克隆
let name = user.name.clone();

// 好：使用引用
let name = &user.name;

// 好：需要所有权时再克隆
let name = user.name.clone(); // 确实需要所有权
```

### 2. 使用适当的集合类型
```
// 小集合使用Vec
let small_list: Vec<i32> = vec![1, 2, 3];

// 需要快速查找使用HashMap
let mut map: HashMap<String, User> = HashMap::new();

// 需要有序集合使用BTreeMap
let mut ordered_map: BTreeMap<String, User> = BTreeMap::new();
```

### 3. 避免字符串分配
```
// 不好：多次分配
let full_name = first_name.to_string() + " " + &last_name;

// 好：使用format!
let full_name = format!("{} {}", first_name, last_name);

// 更好：需要时再分配
fn greet(name: &str) -> String {
    format!("Hello, {}!", name)
}
```

### 4. 合理使用异步
```
// 不好：阻塞异步运行时
let data = blocking_operation().await; // 阻塞操作

// 好：使用spawn_blocking
let data = tokio::task::spawn_blocking(|| {
    blocking_operation()
}).await?;
```

## 安全规范

### 1. 输入验证
```
pub fn validate_username(username: &str) -> Result<(), ValidationError> {
    if username.is_empty() {
        return Err(ValidationError::Empty);
    }
    
    if username.len() > 50 {
        return Err(ValidationError::TooLong(50));
    }
    
    // 只允许字母、数字、下划线
    if !username.chars().all(|c| c.is_alphanumeric() || c == '_') {
        return Err(ValidationError::InvalidChars);
    }
    
    Ok(())
}
```

### 2. SQL注入防护
```
// 使用参数化查询
let user = sqlx::query_as!(
    User,
    "SELECT * FROM users WHERE id = $1",
    user_id
)
.fetch_optional(pool)
.await?;

// 使用ORM
let user = User::find_by_id(user_id)
    .one(pool)
    .await?;
```

### 3. 密码安全
```
use argon2::{Argon2, PasswordHasher, PasswordVerifier};
use argon2::password_hash::{SaltString, PasswordHash};

pub async fn hash_password(password: &str) -> Result<String, Error> {
    let salt = SaltString::generate(&mut rand::thread_rng());
    let argon2 = Argon2::default();
    let password_hash = argon2.hash_password(password.as_bytes(), &salt)?;
    Ok(password_hash.to_string())
}

pub async fn verify_password(password: &str, hash: &str) -> Result<bool, Error> {
    let parsed_hash = PasswordHash::new(hash)?;
    let argon2 = Argon2::default();
    Ok(argon2.verify_password(password.as_bytes(), &parsed_hash).is_ok())
}
```

## 代码审查要点

### 必须检查项
1. 代码格式是否符合`cargo fmt`
2. 是否有编译器警告（`cargo clippy`）
3. 错误处理是否完整
4. 测试是否覆盖核心逻辑
5. 安全漏洞（SQL注入、XSS等）

### 建议检查项
1. 代码可读性
2. 性能优化机会
3. 代码复用性
4. 文档完整性

### 常见问题
1. 不必要的克隆
2. 未处理的错误
3. 硬编码的魔法值
4. 过长的函数
5. 复杂的嵌套

### 代码示例模板

#### 服务层示例
```
/// 用户服务
#[derive(Clone)]
pub struct UserService {
    repository: Arc<dyn UserRepository>,
}

impl UserService {
    pub fn new(repository: Arc<dyn UserRepository>) -> Self {
        Self { repository }
    }
    
    pub async fn get_user(&self, user_id: i64) -> Result<User, ServiceError> {
        let user = self.repository
            .find_by_id(user_id)
            .await?
            .ok_or(ServiceError::UserNotFound(user_id))?;
        
        Ok(user)
    }
}
```

pub async fn get_user(

/// 认证中间件
pub async fn auth_middleware<B>(
    mut request: Request<B>,
    next: Next<B>,
) -> Result<Response, AppError> {
    let auth_header = request.headers()
        .get(header::AUTHORIZATION)
        .and_then(|h| h.to_str().ok());
    
    match auth_header {
        Some(token) => {
            let claims = verify_token(token)?;
            request.extensions_mut().insert(claims);
            Ok(next.run(request).await)
        }
        None => Err(AppError::Unauthorized),
    }
}
