## 用户输入

```
$ARGUMENTS
```

## 目标

执行Rust项目代码规范检查，包括：
1. cargo clippy静态代码检查
2. cargo fmt代码格式化检查
3. 项目编码规范扫描（命名、注释、类型定义等）
4. 输出详细的检查报告

## 执行步骤

### 1. 确认项目环境

首先确认当前目录是Rust项目根目录，检查以下文件是否存在：
- `Cargo.toml`
- `src/` 目录结构
- `.rustfmt.toml` 或 `rustfmt.toml`（可选）

如果不存在，输出错误信息并终止。

### 2. 执行cargo clippy检查

运行以下命令执行clippy检查：

```
cargo clippy -- -D warnings
```

捕获输出结果，分析违规项。

### 3. 执行cargo fmt检查

运行以下命令检查代码格式：

```
cargo fmt --check
```

检查代码格式是否符合规范。

### 4. 代码规范扫描

根据 `rules/项目编码规范.md` 定义的规范，扫描代码检查以下内容：

#### 禁止事项检查
- Tab缩进检查（必须使用4空格）
- 行尾空格检查
- 未使用的导入检查
- 未使用的变量检查
- 魔法数字检查
- 过长的函数检查（>80行）
- 过深的嵌套检查（>3层）

#### 命名规范检查
- 结构体命名检查（大驼峰：UserProfile）
- 枚举命名检查（大驼峰：UserRole）
- trait命名检查（大驼峰：Repository）
- 变量命名检查（蛇形命名：user_name）
- 常量命名检查（全大写下划线：MAX_RETRY_COUNT）
- 模块命名检查（小写字母下划线：user_service）

#### 注释规范检查
- 公共API文档注释检查（必须包含///注释）
- 模块文档注释检查（必须包含//!注释）
- 复杂逻辑注释检查
- TODO/FIXME注释检查

#### 类型定义检查
- 结构体字段文档检查
- 枚举变体文档检查
- 错误类型定义检查
- 生命周期参数检查

## 检查规则参考

### 禁止事项

| 规则 | 说明 | 严重级别 |
|------|------|----------|
| 禁止Tab缩进 | 统一使用4空格 | ERROR |
| 禁止行尾空格 | 行尾不能有空格 | ERROR |
| 禁止未使用导入 | 移除未使用的use语句 | WARN |
| 禁止未使用变量 | 使用_前缀或移除 | WARN |
| 禁止魔法数字 | 定义为常量 | WARN |
| 禁止过长函数 | 函数≤80行（建议） | WARN |
| 禁止过深嵌套 | 嵌套≤3层（建议） | WARN |

### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 结构体 | 大驼峰 | UserProfile, DatabaseConfig |
| 枚举 | 大驼峰 | UserRole, ErrorCode |
| trait | 大驼峰 | Repository, Service |
| 变量 | 蛇形命名 | user_name, max_retries |
| 常量 | 全大写+下划线 | MAX_RETRY_COUNT, DEFAULT_TIMEOUT |
| 模块 | 小写+下划线 | user_service, api_handler |

### 注释规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 公共函数 | /// 文档注释 | /// 获取用户信息 |
| 模块 | //! 模块注释 | //! 用户管理模块 |
| 复杂逻辑 | // 行内注释 | // 检查用户权限 |
| TODO/FIXME | 明确标注 | // TODO: 优化查询性能 |

## 使用示例

```
# 检查全项目
/code-style-srv-check

# 检查指定模块
/code-style-srv-check src/api

# 检查指定文件
/code-style-srv-check src/domain/user/model.rs
```

## 注意事项

1. 检查前确保Rust工具链已安装：`rustup update`
2. 需要安装clippy：`rustup component add clippy`
3. 需要安装rustfmt：`rustup component add rustfmt`
4. 检查结果仅输出到命令行，不生成文件
5. 建议在代码提交前执行此检查
6. 遵循AGENTS.md中的分层架构原则
