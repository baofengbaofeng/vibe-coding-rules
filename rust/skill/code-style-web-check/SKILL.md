## 用户输入

```
$ARGUMENTS
```

## 目标

执行Rust Web前端代码规范检查，包括：
1. 前端框架规范检查（Leptos/Yew组件规范）
2. 组件设计检查（Props、State、生命周期）
3. 样式与布局检查（CSS/Tailwind使用）
4. 前端安全检查（XSS防护、CSRF防护）
5. 性能优化检查（组件渲染优化、资源加载）
6. 输出详细的前端代码规范报告

## 执行步骤

### 1. 确认项目环境

首先确认当前目录是Rust Web前端项目根目录，检查以下技术栈：
- Rust 1.75+
- 前端框架：Leptos 0.5+ 或 Yew 0.20+
- 样式方案：Tailwind CSS 或 自定义CSS
- 构建工具：Trunk 或 wasm-pack
- 状态管理：Signals（Leptos）或 Redux模式

### 2. 前端框架规范检查

#### Leptos框架检查
- 检查组件函数使用#[component]属性
- 检查响应式信号使用（create_signal, create_memo）
- 检查资源加载使用（create_resource）
- 检查服务端渲染配置（SSR）

#### Yew框架检查
- 检查组件结构实现Component trait
- 检查属性定义使用#[derive(Properties)]
- 检查消息处理使用Callback
- 检查生命周期方法实现

### 3. 组件设计检查

#### Props设计检查
- 检查Props类型定义是否明确
- 检查默认值设置是否合理
- 检查可选参数使用Option<T>
- 检查Props验证逻辑

#### State管理检查
- 检查状态提升是否合理
- 检查状态更新性能
- 检查状态持久化
- 检查状态派生计算

#### 生命周期检查
- 检查组件挂载/卸载逻辑
- 检查副作用管理（use_effect）
- 检查清理函数实现
- 检查依赖数组设置

### 4. 样式与布局检查

#### CSS使用检查
- 检查CSS模块化使用
- 检查样式命名规范（BEM或其他）
- 检查响应式设计实现
- 检查CSS变量使用

#### Tailwind检查
- 检查Tailwind类名使用规范
- 检查自定义配置扩展
- 检查暗色模式支持
- 检查动画过渡效果

### 5. 前端安全检查

#### XSS防护检查
- 检查用户输入转义
- 检查innerHTML使用风险
- 检查URL参数验证
- 检查富文本编辑器安全

#### CSRF防护检查
- 检查CSRF令牌使用
- 检查同源策略配置
- 检查CORS设置
- 检查API请求安全头

### 6. 性能优化检查

#### 组件渲染优化
- 检查不必要的重新渲染
- 检查Memoization使用
- 检查虚拟列表实现
- 检查懒加载组件

#### 资源加载优化
- 检查代码分割配置
- 检查图片懒加载
- 检查字体优化
- 检查缓存策略

## 检查规则参考

### 前端框架规范

| 框架 | 规范要求 | 检查级别 |
|------|----------|----------|
| Leptos | 使用#[component]属性 | ERROR |
| Leptos | 响应式信号正确使用 | ERROR |
| Yew | 实现Component trait | ERROR |
| Yew | 使用#[derive(Properties)] | ERROR |

### 组件设计规范

| 检查项 | 规范要求 | 检查级别 |
|--------|----------|----------|
| Props类型 | 明确类型定义 | ERROR |
| 状态管理 | 合理状态提升 | WARN |
| 生命周期 | 完整生命周期方法 | ERROR |
| 副作用 | 正确管理副作用 | ERROR |

### 前端安全规范

| 检查项 | 安全要求 | 风险等级 |
|--------|----------|----------|
| XSS防护 | 用户输入转义 | CRITICAL |
| CSRF防护 | 使用CSRF令牌 | HIGH |
| CORS配置 | 合理同源策略 | MEDIUM |
| 安全头 | 设置安全HTTP头 | MEDIUM |

## 使用示例

```
# 全项目前端检查
/code-style-web-check

# 检查组件规范
/code-style-web-check components

# 检查页面性能
/code-style-web-check pages
```

## 注意事项

1. 前端检查需要结合具体框架（Leptos/Yew）
2. 性能优化要考虑WASM包大小和加载时间
3. 建议在开发过程中持续进行前端检查
4. 新组件开发必须通过前端规范检查
5. 遵循AGENTS.md中的前端架构原则
6. 监控Web Vitals指标并持续优化
