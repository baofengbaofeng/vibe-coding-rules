## 用户输入

```
$ARGUMENTS
```

## 目标

执行前端项目代码检查，包括：
1. Vue 3代码规范检查（Composition API使用、组件规范、模板语法）
2. TypeScript规范检查（类型安全、严格模式、禁止any类型）
3. 组件设计检查（单一职责、可复用性、可测试性）
4. 状态管理检查（Pinia使用规范、状态设计）
5. 性能优化检查（代码分割、懒加载、图片优化）
6. 输出详细的前端代码检查报告

## 执行步骤

### 1. 确认项目环境

首先确认当前目录是前端项目根目录（`src/web/`），检查以下技术栈：
- Vue 3 + Composition API
- TypeScript 5.0+（严格模式）
- Vant 4（UI组件库）
- Pinia 2.0+（状态管理）
- Vite 5.0+（构建工具）
- pnpm 8.0+（包管理器）

### 2. Vue 3代码规范检查

#### Composition API使用检查
- 检查是否使用Composition API + `<script setup>`
- 检查是否使用Options API（禁止使用）
- 检查Composition API使用规范

#### 组件规范检查
- 检查单文件组件结构是否规范
- 检查Props定义是否使用TypeScript
- 检查Emits定义是否规范
- 检查组件暴露（defineExpose）是否合理

#### 模板语法检查
- 检查模板语法是否规范
- 检查指令使用是否合理
- 检查事件处理是否规范

### 3. TypeScript规范检查

#### 类型安全检查
- 检查是否使用any类型（禁止使用）
- 检查类型定义是否完整
- 检查泛型使用是否合理
- 检查接口和类型别名使用

#### 严格模式检查
- 检查tsconfig.json是否开启严格模式
- 检查strictNullChecks是否开启
- 检查noImplicitAny是否开启

#### 类型定义检查
- 检查全局类型定义是否规范
- 检查组件类型定义是否完整
- 检查API类型定义是否规范

### 4. 组件设计检查

#### 单一职责检查
- 检查组件是否职责单一
- 检查组件是否过于复杂
- 检查组件拆分是否合理

#### 可复用性检查
- 检查组件是否设计为可复用的
- 检查Slot使用是否合理
- 检查Provide/Inject使用是否规范

#### 可测试性检查
- 检查组件是否易于测试
- 检查组件依赖是否可Mock
- 检查组件接口是否清晰

### 5. 状态管理检查

#### Pinia使用检查
- 检查是否使用Pinia（禁止使用Vuex）
- 检查Store定义是否规范
- 检查状态设计是否合理

#### 状态设计检查
- 检查状态是否可序列化
- 检查状态更新是否规范
- 检查状态持久化是否合理

### 6. 性能优化检查

#### 代码分割检查
- 检查路由是否使用懒加载
- 检查组件是否使用异步加载
- 检查代码分割策略是否合理

#### 资源优化检查
- 检查图片是否使用懒加载
- 检查是否使用WebP格式
- 检查静态资源优化

## 检查规则参考

### Vue 3规范规则

| 检查项 | 规范要求 | 严重级别 |
|--------|----------|----------|
| Composition API | 必须使用，禁止Options API | HIGH |
| `<script setup>` | 必须使用 | HIGH |
| Props定义 | 必须使用TypeScript接口 | MEDIUM |
| Emits定义 | 必须规范定义 | MEDIUM |

### TypeScript规范规则

| 检查项 | 规范要求 | 严重级别 |
|--------|----------|----------|
| any类型 | 禁止使用 | HIGH |
| 严格模式 | 必须开启 | HIGH |
| 类型定义 | 必须完整 | MEDIUM |
| 泛型使用 | 合理使用 | MEDIUM |

### 组件设计规则

| 检查项 | 规范要求 | 严重级别 |
|--------|----------|----------|
| 单一职责 | 组件职责单一 | HIGH |
| 可复用性 | 设计为可复用的 | MEDIUM |
| 可测试性 | 易于测试 | MEDIUM |
| 组件大小 | ≤500行（建议） | MEDIUM |

### 状态管理规则

| 检查项 | 规范要求 | 严重级别 |
|--------|----------|----------|
| Pinia使用 | 必须使用，禁止Vuex | HIGH |
| 状态设计 | 合理设计，可序列化 | MEDIUM |
| 状态更新 | 规范更新 | MEDIUM |

## 使用示例

```
# 全前端项目检查
/frontend-code-check

# 检查指定组件目录
/frontend-code-check src/web/components

# 检查视图组件
/frontend-code-check views
```

## 注意事项

1. 前端检查需要在 `src/web/` 目录下执行
2. 检查前确保依赖已安装：`pnpm install`
3. 新组件开发必须遵循前端编码规范
4. 定期进行前端代码审查
5. 遵循AGENTS.md中的前端技术栈要求
