## 用户输入

```
$ARGUMENTS
```

## 目标

执行Python项目代码安全检查，包括：
1. 注入防护检查（SQL注入、命令注入、XPath注入）
2. 敏感信息检查（硬编码密钥、配置泄露、日志脱敏）
3. 访问控制检查（API鉴权、权限校验、CORS配置）
4. 输入验证检查（参数验证、文件上传验证）
5. 输出详细的安全检查报告

## 执行步骤

### 1. 确认项目环境

首先确认当前目录是Python项目根目录，检查以下技术栈：
- Python 3.12+
- Tornado 6.3+ 或 FastAPI
- SQLAlchemy 2.0+
- Pydantic 2.0+

### 2. 注入防护检查

#### SQL注入检查
- 检查SQL语句是否使用字符串拼接
- 检查是否使用SQLAlchemy参数绑定
- 检查动态表名/列名是否进行白名单校验

#### 命令注入检查
- 检查是否使用 `os.system()`、`subprocess.run()` 执行用户输入
- 检查命令执行是否使用参数列表而非字符串拼接

#### XPath注入检查
- 检查XPath查询是否参数化处理

### 3. 敏感信息检查

#### 硬编码密钥检查
- 检查代码中是否硬编码数据库密码、Redis密码、JWT密钥等
- 检查是否使用环境变量或加密存储敏感配置

#### 配置泄露检查
- 检查 `.env` 文件是否包含敏感信息
- 检查配置文件是否提交到版本控制

#### 日志脱敏检查
- 检查日志输出是否包含身份证、手机号、银行卡等敏感信息
- 检查敏感字段是否进行脱敏处理

### 4. 访问控制检查

#### API鉴权检查
- 检查所有API是否经过认证中间件验证
- 检查是否有新增的无鉴权接口
- 检查认证中间件是否正确实现

#### 权限校验检查
- 检查业务层是否进行数据权限校验
- 检查是否可能发生越权访问

#### CORS配置检查
- 检查CORS是否使用白名单而非允许所有来源
- 检查CORS中间件是否正确配置

### 5. 输入验证检查

#### 参数验证检查
- 检查是否使用Pydantic进行严格的输入验证
- 检查验证规则是否完整（长度、格式、范围等）

#### 文件上传验证检查
- 检查文件上传是否验证类型和大小
- 检查是否使用白名单限制文件类型

### 6. 安全配置检查

#### 安全响应头检查
- 检查是否设置安全相关的HTTP响应头
- 检查生产环境是否强制HTTPS

#### 速率限制检查
- 检查是否配置速率限制防止暴力破解

## 检查规则参考

### 注入防护规则

| 检查项 | 安全要求 | 风险等级 |
|--------|----------|----------|
| SQL注入 | 使用参数绑定，禁止字符串拼接 | CRITICAL |
| 命令注入 | 使用参数列表，禁止shell拼接 | HIGH |
| XPath注入 | 使用变量解析，禁止字符串拼接 | MEDIUM |

### 敏感信息规则

| 检查项 | 安全要求 | 风险等级 |
|--------|----------|----------|
| 硬编码密钥 | 使用环境变量或加密存储 | HIGH |
| 配置泄露 | 敏感配置不提交版本控制 | MEDIUM |
| 日志脱敏 | 敏感字段脱敏后输出 | MEDIUM |

### 访问控制规则

| 检查项 | 安全要求 | 风险等级 |
|--------|----------|----------|
| API鉴权 | 所有API必须经过认证 | HIGH |
| 权限校验 | 业务层数据权限校验 | HIGH |
| CORS配置 | 使用白名单，禁止所有来源 | MEDIUM |

## 使用示例

```
# 全项目安全检查
/code-security-check

# 检查认证相关模块
/code-security-check auth

# 检查指定处理器
/code-security-check src/app/handlers
```

## 注意事项

1. 安全检查需要结合具体业务场景
2. 安全风险等级根据业务影响评估
3. 建议定期执行安全检查（每周/每月）
4. 新功能开发必须通过安全检查
5. 遵循AGENTS.md中的安全优先原则
