## 用户输入

```
$ARGUMENTS
```

## 目标

执行Python项目代码规范检查，包括：
1. flake8静态代码检查
2. 项目编码规范扫描（命名、注释、导入、异常处理等）
3. 输出详细的检查报告

## 执行步骤

### 1. 确认项目环境

首先确认当前目录是Python项目根目录，检查以下文件是否存在：
- `pyproject.toml` 或 `setup.py` 或 `requirements.txt`
- `src/` 目录结构

如果不存在，输出错误信息并终止。

### 2. 执行flake8检查

运行以下命令执行flake8检查：

```
flake8 src/
```

捕获输出结果，分析违规项。

### 3. 代码规范扫描

根据 `rules/后端编码规范.md` 定义的规范，扫描代码检查以下内容：

#### 禁止事项检查
- Tab缩进检查（必须使用4空格）
- 行尾空格检查
- `print()` 使用检查（必须使用logging模块）
- 裸 `Exception` 捕获检查
- `from module import *` 检查
- 魔法数字检查
- 全局变量检查

#### 命名规范检查
- 类命名检查（大驼峰 + 后缀：Controller、Module、Entity、Service等）
- 方法命名检查（小驼峰 + 动词前缀：query_user_by_id、create_user等）
- 常量命名检查（全大写下划线：MAX_RETRY_COUNT）
- 变量命名检查（小驼峰，禁止二义性缩写）
- 模块命名检查（小写字母下划线：user_module、user_controller）

#### 注释规范检查
- 类Docstring检查（必须包含模块说明、作者、创建时间）
- 公共方法Docstring检查（必须包含参数说明、返回值说明、异常说明）
- 字段注释检查

#### 导入规范检查
- 导入顺序检查（标准库 -> 第三方库 -> 项目内部）
- 禁止星号导入检查

## 检查规则参考

### 禁止事项

| 规则 | 说明 | 严重级别 |
|------|------|----------|
| 禁止Tab缩进 | 统一使用4空格 | ERROR |
| 禁止行尾空格 | 行尾不能有空格 | ERROR |
| 禁止print()调试 | 使用logging模块 | ERROR |
| 禁止裸Exception | 捕获具体异常类型 | ERROR |
| 禁止星号导入 | 每行导入一个模块/类 | ERROR |
| 禁止魔法数字 | 定义为常量 | WARN |
| 禁止全局变量 | 使用类属性或配置 | WARN |

### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 处理器 | XxxHandler | UserHandler |
| 领域服务 | XxxService | UserService |
| 数据实体 | XxxEntity | UserEntity |
| 仓储 | XxxRepository | UserRepository |
| 工具类 | XxxUtil | StringUtil |
| 常量 | SCREAMING_SNAKE_CASE | MAX_RETRY_COUNT |
| 变量 | 小驼峰 | user_identifier |

### 方法命名前缀

| 操作 | 前缀 | 示例 |
|------|------|------|
| 查询 | query/get/find | query_user_by_id |
| 创建 | create/add | create_user |
| 更新 | update/modify | update_user |
| 删除 | delete/remove | delete_user |
| 验证 | validate/check | validate_email |
| 处理 | process/handle | process_request |

## 使用示例

```
# 检查全项目
/code-style-check

# 检查指定文件
/code-style-check src/app/handlers/users.py

# 检查指定模块
/code-style-check domain
```

## 注意事项

1. 检查前确保Python环境为3.12+
2. 需要安装flake8：`pip install flake8`
3. 检查结果仅输出到命令行，不生成文件
4. 建议在代码提交前执行此检查
5. 遵循AGENTS.md中的分层架构原则
