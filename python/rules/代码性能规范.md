# 代码性能规范.md

本规范定义代码编写过程中的性能优化标准，涵盖数据库操作、缓存使用、并发控制、循环处理、IO操作等方面，确保系统高效稳定运行。

## 数据库性能

### 查询优化

- **禁止全表扫描**：WHERE条件必须命中索引
- **禁止`SELECT *`**：必须指定需要的字段列表
- **复杂查询需EXPLAIN分析**：执行计划中type避免出现ALL
- **分页查询必须带排序**：避免分页结果不稳定

```
# 禁止
query = select(UserEntity)

# 推荐
query = select(
    UserEntity.id,
    UserEntity.username,
    UserEntity.email
).where(
    UserEntity.status == 1
).order_by(
    UserEntity.created_at.desc()
).limit(100)
```

### 索引规范

- 避免在索引列上使用函数或计算
- 单表索引数量不超过5个
- 联合索引遵循最左前缀原则

```
# 禁止：索引列使用函数
query = select(UserEntity).where(
    func.date(UserEntity.created_at) == "2026-04-17"
)

# 推荐：范围查询
query = select(UserEntity).where(
    UserEntity.created_at >= datetime(2026, 4, 17),
    UserEntity.created_at < datetime(2026, 4, 18)
)
```

### 批量操作

- **禁止循环执行SQL**：必须使用批量操作
- 批量插入单次不超过1000条
- 批量更新使用批量语句

```
# 禁止：循环插入
for user in users:
    session.add(user)
await session.commit()

# 推荐：批量插入
session.add_all(users)
await session.commit()

# 推荐：批量更新
stmt = update(UserEntity).where(
    UserEntity.id == bindparam("user_id")
).values(
    status=bindparam("status")
)
await session.execute(stmt, [
    {"user_id": 1, "status": 1},
    {"user_id": 2, "status": 1},
])
```

## 缓存性能

### Redis使用规范

- **大Key拆分**：单个Key的Value不超过10KB
- **Key必须设置过期时间**：避免内存泄漏
- **Key必须设置业务前缀**：如 `project:dev:user:1001`
- **禁止使用KEYS命令**：生产环境使用SCAN

```
import redis.asyncio as redis
from datetime import timedelta

# 推荐：带前缀和过期时间
redis_client = redis.from_url(settings.redis_url)

async def cache_user(user_id: int, user_data: dict):
    key = f"project:user:{user_id}"
    await redis_client.setex(
        key,
        timedelta(minutes=30),
        json.dumps(user_data)
    )
```

### 缓存策略

- **缓存穿透**：查询空结果也缓存，设置较短过期时间
- **缓存击穿**：热点数据使用分布式锁或永不过期
- **缓存雪崩**：过期时间加随机值，避免同时失效

```
import asyncio
from app.infrastructure.cache.redis_client import redis_client

async def get_user_with_cache(user_id: int):
    key = f"project:user:{user_id}"
    
    # 尝试从缓存获取
    cached = await redis_client.get(key)
    if cached is not None:
        if cached == "NULL":  # 空结果缓存
            return None
        return json.loads(cached)
    
    # 缓存击穿防护：分布式锁
    lock_key = f"project:lock:user:{user_id}"
    lock = redis_client.lock(lock_key, timeout=10)
    
    try:
        # 尝试获取锁
        if await lock.acquire(blocking_timeout=1):
            # 再次检查缓存（双重检查）
            cached = await redis_client.get(key)
            if cached is not None:
                if cached == "NULL":
                    return None
                return json.loads(cached)
            
            # 查询数据库
            user = await user_repository.get_by_id(user_id)
            
            # 写入缓存
            if user is None:
                # 空结果缓存，设置较短过期时间
                await redis_client.setex(key, timedelta(minutes=5), "NULL")
            else:
                # 正常缓存，过期时间加随机值避免雪崩
                expire_seconds = 1800 + random.randint(-300, 300)  # 30分钟±5分钟
                await redis_client.setex(
                    key,
                    expire_seconds,
                    json.dumps(user.dict())
                )
            
            return user
        else:
            # 等待其他线程/进程完成
            await asyncio.sleep(0.1)
            return await get_user_with_cache(user_id)
    finally:
        try:
            await lock.release()
        except:
            pass
```

## 并发控制

### Tornado异步编程

- **使用Tornado异步**：IO密集型操作必须使用Tornado异步
- **避免阻塞IOLoop**：CPU密集型操作使用线程池
- **合理使用async/await**：避免不必要的异步

```
import tornado.ioloop
from tornado.concurrent import run_on_executor
from concurrent.futures import ThreadPoolExecutor

class AsyncService:
    """异步服务基类"""
    
    def __init__(self):
        self.executor = ThreadPoolExecutor(max_workers=4)
    
    @run_on_executor
    def cpu_intensive_task(self, data):
        """CPU密集型任务（在线程池中执行）"""
        return cpu_intensive_function(data)
    
    async def io_intensive_task(self, url):
        """IO密集型任务（在IOLoop中执行）"""
        http_client = tornado.httpclient.AsyncHTTPClient()
        try:
            response = await http_client.fetch(url)
            return response.body
        except tornado.httpclient.HTTPError as e:
            logger.error(f"HTTP request failed: {e}")
            return None

# 使用示例
service = AsyncService()

# CPU密集型任务
result = await service.cpu_intensive_task(data)

# IO密集型任务
result = await service.io_intensive_task(url)
```

### 分布式锁

- **必须使用redis-py实现**：禁止自行实现分布式锁
- **必须设置超时时间**：避免死锁
- **锁粒度要小**：避免大范围锁定

```
import redis.asyncio as redis
from contextlib import asynccontextmanager

@asynccontextmanager
async def distributed_lock(redis_client: redis.Redis, lock_key: str, timeout: int = 10):
    lock = redis_client.lock(lock_key, timeout=timeout)
    acquired = False
    
    try:
        acquired = await lock.acquire(blocking_timeout=5)
        if acquired:
            yield lock
        else:
            raise TimeoutError(f"Failed to acquire lock: {lock_key}")
    finally:
        if acquired:
            await lock.release()

# 使用示例
async def update_user_balance(user_id: int, amount: float):
    lock_key = f"project:lock:user_balance:{user_id}"
    
    async with distributed_lock(redis_client, lock_key) as lock:
        # 执行业务逻辑
        balance = await get_user_balance(user_id)
        new_balance = balance + amount
        await update_balance(user_id, new_balance)
```

## 循环优化

### 循环内禁止操作

- **禁止循环内执行数据库操作**：提取到循环外批量处理
- **禁止循环内调用远程接口**：改为批量接口或并行调用
- **禁止循环内创建大量对象**：考虑对象复用

```
# 禁止：循环内查库
user_ids = [1, 2, 3, 4, 5]
users = []
for user_id in user_ids:
    user = await user_repository.get_by_id(user_id)
    users.append(user)

# 推荐：批量查询
user_ids = [1, 2, 3, 4, 5]
users = await user_repository.get_by_ids(user_ids)
user_map = {user.id: user for user in users}

for user_id in user_ids:
    user = user_map.get(user_id)
    # 处理逻辑
```

### 循环优化技巧

- **减少循环次数**：提前过滤无效数据
- **避免循环内重复计算**：提取到循环外
- **合理使用生成器**：大数据集使用生成器避免内存占用

```
# 禁止：循环内重复计算
for user in users:
    if user.created_at > calculate_deadline():
        # 处理逻辑

# 推荐：提取计算
deadline = calculate_deadline()
for user in users:
    if user.created_at > deadline:
        # 处理逻辑

# 推荐：使用生成器
def filter_active_users(users):
    for user in users:
        if user.is_active():
            yield user

active_users = list(filter_active_users(users))
```

### 集合遍历

- **遍历时避免修改集合**：创建副本或使用列表推导
- **根据场景选择合适集合**：set查找快，list顺序访问快
- **大集合分批处理**：避免内存溢出

from itertools import islice

# 推荐：分批处理大集合
def batch_process(items, batch_size=1000):

        if not batch:
            break
        process_batch(batch)

## IO性能

### 文件操作

- **使用缓冲读写**：避免频繁系统调用
- **必须关闭资源**：使用with语句
- **避免频繁打开关闭**：批量操作时复用文件句柄

```
# 推荐：缓冲读写
import aiofiles

async def read_large_file(file_path: str):
    async with aiofiles.open(file_path, "r") as f:
        async for line in f:  # 逐行读取，避免内存溢出
            process_line(line)

# 推荐：批量写入
async def write_batch_data(file_path: str, data_list: list):
    async with aiofiles.open(file_path, "a") as f:
        batch_content = "\n".join(str(item) for item in data_list)
        await f.write(batch_content + "\n")
```

### 网络请求

- **设置合理超时**：连接超时、读取超时
- **使用连接池**：HTTP连接池复用
- **批量操作并行处理**：asyncio.gather并行调用

```
import asyncio
import aiohttp

async def fetch_multiple_urls(urls: list[str]):
    async with aiohttp.ClientSession(
        timeout=aiohttp.ClientTimeout(total=30)
    ) as session:
        tasks = []
        for url in urls:
            task = fetch_url(session, url)
            tasks.append(task)
        
        # 并行执行
        results = await asyncio.gather(*tasks, return_exceptions=True)
        return results

async def fetch_url(session: aiohttp.ClientSession, url: str):
    try:
        async with session.get(url) as response:
            return await response.json()
    except Exception as e:
        logger.error(f"Failed to fetch {url}: {e}")
        return None
```

### 序列化

- **选择高效序列化方式**：JSON通用性好，MessagePack性能高
- **避免序列化大对象**：按需传输字段
- **使用orjson替代json**：性能更好

```
import orjson

def serialize_user(user):

### 生成器与迭代器

```
# 推荐：使用生成器处理大数据集
def read_large_file_generator(file_path):
    with open(file_path, "r") as f:
        for line in f:
            yield line.strip()

# 处理时逐行处理，不加载到内存
for line in read_large_file_generator("large_file.txt"):

### 避免内存泄漏

- **及时关闭资源**：数据库连接、文件句柄、网络连接
- **循环引用处理**：使用weakref避免循环引用
- **缓存清理**：定期清理过期缓存

```
import weakref

# 使用weakref避免循环引用
class DataHandler:
    def __init__(self, data):
        self.data = data
        self._callback = None
    
    def set_callback(self, callback):
        # 使用weakref避免循环引用
        self._callback = weakref.ref(callback)
    
    def process(self):
        if self._callback:
            callback = self._callback()
            if callback:
                callback(self.data)
```

## 性能检测

### 性能监控

```
import time
from contextlib import contextmanager

@contextmanager
def measure_time(operation_name: str):
    start_time = time.perf_counter()
    try:
        yield
    finally:
        elapsed = time.perf_counter() - start_time
        logger.info(f"{operation_name} took {elapsed:.3f} seconds")
        if elapsed > 1.0:  # 慢操作警告
            logger.warning(f"Slow operation: {operation_name}")

# 使用示例
with measure_time("database_query"):
    result = await database_query()
```
