# 单元测试规范.md

本规范定义项目单元测试的编写标准，确保测试代码质量、可维护性和覆盖率要求。

---

## 技术栈

- **测试框架**: pytest 7.4+
- **Mock框架**: pytest-mock 3.12+
- **断言库**: pytest内置断言 + pytest-assume（复杂断言）
- **覆盖率工具**: pytest-cov 4.1+
- **Tornado测试**: pytest-tornado 2.0+

---

### 命名规范

- **测试文件命名**: `test_模块名.py` 或 `模块名_test.py`
- **测试类命名**: `Test类名`，如 `TestUserService`
- **测试方法命名**: `test_描述测试场景`，如：
  - `test_should_return_user_when_exists`（正向场景）
  - `test_should_raise_exception_when_user_not_found`（异常场景）
  - `test_is_accessible_when_user_status_is_active`（条件场景）

## 编写规范

### 基础约束

1. **必须隔离外部依赖**：测试不得依赖数据库、Redis、外部服务等，必须使用Mock
2. **必须包含明确断言**：每个测试方法必须有至少一个断言
3. **禁止启动完整应用**：单元测试应轻量化，禁止启动FastAPI应用
4. **禁止测试私有方法**：只测试公共方法，私有方法通过公共方法间接测试
5. **单测必须可重复执行**：测试之间不能有依赖关系，执行顺序不影响结果

### 测试方法结构

遵循 **Given-When-Then** 模式：

```
def test_should_encrypt_successfully():
    # Given - 准备测试数据和Mock
    crypto_service = mocker.Mock(spec=CryptoService)
    crypto_service.encrypt.return_value = "encrypted_value"
    util = MyUtil(crypto_service)

    # When - 执行被测方法
    result = util.process("test_value")

    # Then - 验证结果
    assert result == "encrypted_value"
```

### Tornado异步测试

Tornado异步测试必须使用 `pytest.mark.gen_test` 装饰器：

```
import pytest
from tornado.testing import gen_test

@pytest.mark.gen_test
async def test_should_fetch_user_async():
    # Given
    user_repo = mocker.AsyncMock(spec=UserRepository)
    user_repo.get_by_id.return_value = UserEntity(id=1, username="test")
    service = UserService(user_repo)

    # When
    result = await service.get_user(1)

    # Then
    assert result.id == 1
    assert result.username == "test"
```

## 分层测试规范

### Tornado处理器测试

处理器测试应验证HTTP请求处理、参数校验、结果封装，禁止验证业务逻辑。

```
import json
import pytest
from tornado.testing import AsyncHTTPTestCase, gen_test
from tornado.web import Application
from app.handlers.v1.users import UserHandler
from app.domain.users.services import UserService

class TestUserHandler(AsyncHTTPTestCase):
    def get_app(self):
        # 创建测试应用
        user_service = self.user_service = mocker.Mock(spec=UserService)
        
        return Application([
            (r"/api/v1/users/([^/]+)", UserHandler, dict(user_service=user_service)),
            (r"/api/v1/users", UserHandler, dict(user_service=user_service)),
        ])
    
    @pytest.mark.gen_test
    async def test_should_create_user_successfully(self):
        # Given
        self.user_service.create_user.return_value = UserResponse(
            id=1, username="test_user", email="test@example.com"
        )
        
        request_data = {
            "username": "test_user",
            "email": "test@example.com",
            "password": "password123"
        }
        
        # When
        response = await self.http_client.fetch(
            self.get_url("/api/v1/users"),
            method="POST",
            body=json.dumps(request_data),
            headers={"Content-Type": "application/json"}
        )
        
        # Then
        assert response.code == 200
        data = json.loads(response.body)
        assert data["data"]["id"] == 1
        assert data["data"]["username"] == "test_user"
        self.user_service.create_user.assert_called_once_with(
            username="test_user",
            email="test@example.com",
            password="password123"
        )
    
    @pytest.mark.gen_test
    async def test_should_return_400_when_validation_fails(self):
        # Given
        invalid_data = {
            "username": "ab",  # 太短
            "email": "invalid-email",
            "password": "123"
        }
        
        # When
        response = await self.http_client.fetch(
            self.get_url("/api/v1/users"),
            method="POST",
            body=json.dumps(invalid_data),
            headers={"Content-Type": "application/json"},
            raise_error=False
        )
        
        # Then
        assert response.code == 400
        data = json.loads(response.body)
        assert "参数验证失败" in data["error"]
```

**要点**：
- 使用 `AsyncHTTPTestCase` 进行集成测试
- Mock所有依赖的Service
- 测试正常流程和异常流程
- 验证HTTP状态码和响应内容

### Service层测试

Service测试验证业务逻辑，这是单元测试的核心区域。

```
class TestUserService:
    def test_should_return_user_when_exists(self, mocker):
        # Given
        user_repo = mocker.Mock(spec=UserRepository)
        user_entity = UserEntity(id=1, username="test_user")
        user_repo.get_by_id.return_value = user_entity
        
        service = UserService(user_repo)
        
        # When
        result = service.get_user(1)
        
        # Then
        assert result.id == 1
        assert result.username == "test_user"
        user_repo.get_by_id.assert_called_once_with(1)

    def test_should_raise_exception_when_user_not_found(self, mocker):
        # Given
        user_repo = mocker.Mock(spec=UserRepository)
        user_repo.get_by_id.return_value = None
        
        service = UserService(user_repo)
        
        # When & Then
        with pytest.raises(UserNotFoundException) as exc_info:
            service.get_user(1)
        
        assert str(exc_info.value) == "用户不存在"
```

**要点**：
- 使用 `mocker.Mock(spec=...)` 创建类型安全的Mock
- Mock所有外部依赖（Repository、远程服务等）
- 验证业务规则的正确性
- 覆盖正常流程和异常流程

### Repository层测试

Repository测试验证数据访问逻辑。

```
class TestUserRepository:
    def test_should_get_user_by_id(self, mocker):
        # Given
        session = mocker.Mock(spec=AsyncSession)
        query = mocker.Mock()
        session.execute.return_value = query
        query.scalar_one_or_none.return_value = UserEntity(id=1, username="test")
        
        repo = UserRepository(session)
        
        # When
        result = repo.get_by_id(1)
        
        # Then
        assert result.id == 1
        assert result.username == "test"
        session.execute.assert_called_once()
```

### 工具类测试

工具类测试应覆盖各种边界情况和异常处理。

```
class TestCryptoUtil:
    def test_should_encrypt_successfully(self, mocker):
        # Given
        crypto_service = mocker.Mock(spec=CryptoService)
        crypto_service.encrypt.return_value = "encrypted_value"
        
        util = CryptoUtil(crypto_service)
        
        # When
        result = util.encrypt("test_value")
        
        # Then
        assert result == "encrypted_value"
        crypto_service.encrypt.assert_called_once_with("test_value")

    def test_should_raise_exception_when_encrypt_fails(self, mocker):
        # Given
        crypto_service = mocker.Mock(spec=CryptoService)
        crypto_service.encrypt.side_effect = RuntimeError("encrypt error")
        
        util = CryptoUtil(crypto_service)
        
        # When & Then
        with pytest.raises(BusinessException) as exc_info:
            util.encrypt("test_value")
        
        assert "加密失败" in str(exc_info.value)

    def test_should_return_empty_when_input_is_none(self):
        # Given
        util = CryptoUtil(mocker.Mock(spec=CryptoService))
        
        # When
        result = util.encrypt(None)
        
        # Then
        assert result == ""
```

**要点**：
- 测试各种输入参数组合
- 测试异常场景
- 测试边界条件（空值、空字符串等）

## Mock规范

### pytest-mock使用

```
import pytest

def test_with_mock(mocker):
    # 创建Mock对象
    mock_service = mocker.Mock(spec=UserService)
    
    # 定义Mock行为
    mock_service.get_user.return_value = UserResponse(id=1)
    
    # 验证方法调用
    mock_service.get_user.assert_called_once_with(1)
    
    # 验证调用次数
    mock_service.get_user.assert_called_once()
    
    # 验证从未被调用
    mock_service.delete_user.assert_not_called()
```

### Tornado异步Mock

```
import pytest
from tornado.testing import gen_test

@pytest.mark.gen_test
async def test_tornado_async_mock(mocker):
    # 创建异步Mock
    mock_repo = mocker.AsyncMock(spec=UserRepository)
    mock_repo.get_by_id.return_value = UserEntity(id=1)
    
    # 调用异步方法
    result = await mock_repo.get_by_id(1)
    
    # 验证
    assert result.id == 1
    mock_repo.get_by_id.assert_awaited_once_with(1)
```

### 参数匹配器

```
def test_with_matchers(mocker):
    mock_service = mocker.Mock(spec=UserService)
    
    # 任意参数
    mock_service.update_user.return_value = True
    mock_service.update_user(mocker.ANY)
    
    # 精确匹配
    mock_service.get_user(1)
    mock_service.get_user.assert_called_with(1)
    
    # 正则匹配
    mock_service.search("test")
    mock_service.search.assert_called_with(mocker.Re(r"test.*"))
```

## 断言规范

### pytest断言

```
# 基本断言
assert result is not None
assert result == expected
assert result is True

# 集合断言
assert len(items) == 3
assert "item" in items
assert all(item > 0 for item in items)

# 异常断言
with pytest.raises(ValueError) as exc_info:
    function_that_raises()
assert "error message" in str(exc_info.value)

# Tornado异步异常断言
import pytest
from tornado.testing import gen_test

@pytest.mark.gen_test
async def test_async_exception():
    with pytest.raises(ValueError):
        await async_function_that_raises()
```

### 复杂断言使用pytest-assume

```
import pytest_assume

def test_complex_assertions():
    result = complex_calculation()
    
    # 多个断言，即使前面的失败也会继续执行后面的
    pytest_assume.assume(result.field1 == expected1)
    pytest_assume.assume(result.field2 == expected2)
    pytest_assume.assume(result.field3 == expected3)
```

### 自定义断言消息

```
def test_with_custom_message():
    result = calculate()
    assert result == expected, f"Expected {expected}, got {result}"
```

## 测试覆盖要求

### 必须覆盖的场景

1. **正常流程**：主要业务逻辑的正常执行路径
2. **异常流程**：参数校验失败、业务异常、系统异常
3. **边界条件**：空值、空集合、最大值、最小值
4. **分支逻辑**：if-else各分支、match各case

### 参数化测试

对于需要多组输入的测试，使用参数化测试：

```
import pytest

@pytest.mark.parametrize("algorithm", ["AES", "DES", "RSA"])
def test_should_support_multiple_algorithms(algorithm):
    # 测试逻辑
    util = CryptoUtil(algorithm=algorithm)
    assert util.is_supported()

@pytest.mark.parametrize("status,expected", [
    (1, True),
    (0, False),
    (-1, False)
])
def test_should_validate_status(status, expected):
    user = UserEntity(status=status)
    assert user.is_active() == expected
```

## 命名与注释

### 测试方法命名

测试方法名应清晰描述测试场景：

```
# 推荐：描述期望行为
def test_should_return_user_when_exists()
def test_should_raise_exception_when_user_not_found()

# 推荐：描述条件判断
def test_is_active_when_status_is_one()
def test_not_active_when_status_is_zero()

# 不推荐：模糊命名
def test1()
def test_user()
```

### 测试类注释

测试类应添加简要注释说明测试范围：

```
"""
用户服务业务逻辑测试。
"""
class TestUserService:
    # ...
```

## 最佳实践

1. **单一职责**：每个测试方法只验证一个行为
2. **快速执行**：单元测试应在秒级完成
3. **可读性**：测试代码应清晰表达测试意图
4. **可维护性**：测试代码应与业务代码同步更新
5. **隔离性**：测试之间完全独立，不共享状态

---

## 禁止事项

1. **禁止启动完整应用**：单元测试不应启动FastAPI应用
2. **禁止依赖外部系统**：数据库、Redis、HTTP服务等必须Mock
3. **禁止使用 `print()` 打印**：测试结果通过断言验证
4. **禁止无断言的测试**：每个测试必须有明确的验证点
5. **禁止测试间依赖**：每个测试方法必须独立可执行
6. **禁止捕获裸 `Exception`**：必须捕获具体异常类型
7. **禁止修改全局状态**：测试不应修改模块级或全局变量
